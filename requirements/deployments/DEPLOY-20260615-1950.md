## Summary

Replaced the unreliable gemma extraction with **Mistral Document AI** (`mistral-document-ai-2505`)
on Azure AI Foundry (EU, no training on data). One OCR call returns the integral
transcription (rawText) + structured extraction (document_annotation via JSON Schema),
PDF native. This solves the long-standing problem where the model omitted clinical
arrays (diagnosi/allergie/farmaci/terapie).

## Changes

- runtime: new `mistral` provider (SDK-free, stdlib urllib); `run_extraction` prefers
  `run_structured`; registered in spec/factory/profiles; `MISTRAL_API_KEY` credential
  in registry; json_schema requires `name` field (422 fix).
- backend: sends the real JSON Schema (loadOutputSchema) to the runtime; serializes the
  runtime's normalized error object to a string (was masked by a Prisma type error).

## Required runtime env (set by user on `clinicos-ai-runtime` service)

- `MISTRAL_OCR_URL=https://dpsaifoundry.services.ai.azure.com/providers/mistral/azure/ocr`
- `MISTRAL_API_KEY=<key>`
- `AI_OCR_MODEL=mistral:mistral-document-ai-2505`
- `AI_EXTRACTION_MODEL=mistral:mistral-document-ai-2505`
