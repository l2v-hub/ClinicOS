# DEPLOY-20260616-0648

## Included requirements

- REQ-025 — Profilo documentale Imola e ordinamento pagine — GitHub Issue #26

## Summary

Adds a configurable, model-agnostic **DocumentProfile** system to the AI runtime
(`clinicos-ai-runtime`). The first profile, `AUSL_IMOLA_DISCHARGE_LETTER_V1`, drives
document classification, page-number recovery (footer first), repetitive header/footer
separation, and deterministic page ordering — with no hospital name hardcoded in the model
or the frontend. Unrecognised documents fall back to `GENERIC_DISCHARGE_LETTER` (never
rejected). Original page text is immutable; clinical/anagraphic/ambiguous lines are never
lost (kept + flagged).

## Safety / scope

Additive package — does not touch the live extraction path (Mistral OCR). No provider SDK
imported (isolation guard passes). Page-text wiring from Mistral's server-side OCR into this
preprocessing is a documented follow-up; the profile system is complete and unit-verified.
