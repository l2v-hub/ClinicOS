# DEPLOY-20260617-0816

## Included requirements

- REQ-036 ‚Äî Consentire ritorno, riordinamento e riprocessamento dei documenti (Critical) ‚Äî GitHub Issue #48

## Summary

The operator can now go back from Elaborazione/Revisione to the editable "documenti" phase,
change the document order, and reprocess WITHOUT re-uploading ‚Äî reusing the same job. A
"‚Üê Torna ai documenti" control is available during processing and review; returning invalidates
the derived draft (after a confirm dialog when a draft exists) and keeps the files. The extraction
now honours the operator-defined order, and a double "Avvia elaborazione" press is idempotent.

## Changes

- backend `job-service.ts`:
  - **`runJob` reads documents `orderBy: sortOrder`** ‚Äî fixes the core bug where the reorder was
    persisted but ignored at extraction time (the operator order is now actually used).
  - new `reopenJob` ‚Äî review_ready/failed/retryable/in-flight ‚Üí `uploaded`, clears `resultData`
    (`Prisma.DbNull`)/`model`/`stage`/`error`; keeps documents + on-disk files.
  - `enqueueJob` made idempotent (already queued/in-flight ‚Üí no-op) ‚Äî double-press guard, on top of
    the single-flight worker's atomic `queued‚Üíprocessing` claim.
  - cooperative abort (`runSuperseded`): an in-flight run that detects a reopen aborts before
    persisting a stale draft.
- backend `routes/ai-jobs.ts`: new `POST /ai/extraction/jobs/:id/reopen`.
- frontend `DischargeImportModal.tsx`: `reopenToDocuments()` + "‚Üê Torna ai documenti" header button
  (shown during Elaborazione and Revisione) + recompute-confirm dialog; Revisione `onBack` rewired
  to reopen (invalidates draft, returns to editable phase). `app-additions.css`: back-button style.

## Limitations

- Per-page OCR reuse is a `clinicos-ai-runtime` (separate service) optimization ‚Äî reprocess here
  reuses uploaded files (no re-upload / no duplicate job) and re-runs the runtime on explicit reprocess.
- Revisione-screen visual evidence requires a live AI runtime (absent locally); covered by unit
  tests + the live API check.
