## Summary

Adds the flat, model-agnostic **discharge narrative contract** (`clinicos-discharge-narrative-v1`):
each clinical section of the discharge letter is one faithful TEXT block (string), not an
array of structured entities. The draft is a deterministic transform of REQ-026's faithful
`_sections`, so by construction it contains **no** `diagnoses[]/medications[]/consultations[]`
— no auto-generated ICD / principale / comorbidità / stato. Semantic emphasis is carried as
offset boldTags (no HTML/Markdown); provenance as sourceReferences.

- `backend/src/ai/sections/narrative.ts` — `DischargeNarrativeDraft` + `buildNarrativeDraft`
  (canonical EN keys → Italian `*Text` fields, home+hospital therapy merged, boldTags with

## Removal of the wrong behaviour

The narrative import output is array-free by design. Pre-existing manual modules are
untouched; only the automatic generation of diagnosis/medication/consultation rows FROM the
discharge letter is removed (the narrative path emits none).
