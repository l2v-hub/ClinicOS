## Summary

Two user-reported issues after the runtime go-live:
1. The import popup had no background (`.modal-card`/`.import-modal` set no surface)
   → transparent over the overlay, unreadable.
2. The review showed only a whitelisted subset of extracted data
   (`mergeExtractions` drops nested groups and non-whitelisted scalars), so the
   operator could not see or integrate all data, nor see the recognized text.

## Fix

- CSS: opaque background + radius + shadow on `.modal-card` / `.import-modal`.
- Backend: `runJob` persists the full raw extraction (`_full`, lossless) + a
  best-effort integral OCR transcription (`rawText`, separate runtime job so it
  cannot break structured extraction). New `GET /ai/extraction/schema`.
- Frontend: new `ImportReviewFull` — schema-driven recursive editor rendering every
  field (anagrafica, all cartella scalars, nested groups, clinical lists with
  add/remove), all editable, plus a searchable OCR text panel. Confirm now sends
  the full reviewed cartella. Old partial `ImportReview` removed.
