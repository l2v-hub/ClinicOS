# DEPLOY-20260615-1101

## Included requirements

- REQ-023 (hotfix #2) — wrap runtime extraction into merged-proposal shape — GitHub Issue #22 — commit d65d05d

## Summary

After the contract fix (DEPLOY-20260615-0126), extraction reached `review_ready`
but the review preview did not render and confirm never fired. The runtime returns
RAW schema JSON (`anagrafica.nome="Mario"`, `cartella.diagnosi=[...]`, no `_merge`),
while the frontend `ImportReview` expects the merged-proposal shape (REQ-016/017):
`proposal._merge.report`, anagrafica as `MergedField {status,value,sources}`,
cartella clinical arrays as `MergedList {items}`. `ImportReview` crashed on
`proposal._merge.report`.

## Fix

`backend/src/ai/upload/job-service.ts`: `runJob` now wraps the runtime's raw
`{anagrafica,cartella}` through the existing `mergeExtractions` layer (new exported
`wrapRuntimeResult()`) before persisting `resultData`, matching the downstream
contract the legacy pipeline already produced. `confirm-service.ts` and the modal
confirm wiring were untouched (already REQ-018-verified).
