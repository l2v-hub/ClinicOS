# DEPLOY-20260615-2158

## Included requirements

- REQ-026 — Estrarre fedelmente le sezioni cliniche e applicare TAG semantici — GitHub Issue #27

## Summary

Adds the **faithful clinical-sections contract** for discharge letters: 12 canonical,
profile-configurable section keys, a model-agnostic JSON Schema + prompt, semantic-tag
annotations (offsets only — never HTML), and an allergy-conflict confirm block. The
section text is preserved verbatim (never summarised, rewritten, reordered or split);
the model output is post-processed deterministically so the contract holds regardless of
which model the runtime uses.

## Changes

- assets: `backend/ai-assets/imola-profile.json` (canonical keys + aliases, editable
  backend-side without any frontend change) and `clinicos-sections.schema.json`.
- module `backend/src/ai/sections/`: `profile.ts` (loader), `prompt.ts` (profile-derived
  faithful-extraction prompt), `validate.ts` (schema check + post-processing: annotation
  offset reconcile/drop, single-block dup-collapse, medication exactText, allergy status,
  no-HTML, `isConfirmBlocked`), `index.ts` (`buildSectionsRequest`).
- backend wiring: opt-in best-effort sections pass in `runJob` (`AI_SECTIONS_PASS`,
  default OFF) storing `resultData._sections`; `confirm-service` blocks confirmation when
  `allergies.status === 'conflicting'` unless `confirmAllergyConflict` override.
- tests: `backend/src/ai/__tests__/sections.test.ts` (20 tests) added to the suite.

## Runtime impact

None required. The AI runtime is generic (prompt + JSON Schema in → JSON out); no
provider/runtime change. The pass is opt-in via `AI_SECTIONS_PASS=true`.
