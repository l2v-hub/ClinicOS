## Summary

After REQ-023/024 went live, **every document import failed** with
"Errore temporaneo durante l'elaborazione. I documenti sono conservati: puoi
riprovare." (job state `retryable_error`).

Root cause: the backend's runtime HTTP client did not match the runtime's neutral
contract (`clinicos-ai-runtime/clinicos_ai/domain/contracts.py`). `runtimeCreateJob`
read `json.id` while the runtime returns `job_id`, so the runtime job id was
`undefined` ‚Üí `POST /v1/document-jobs/undefined/run` returned 404 ‚Üí `provider_error`
‚Üí `retryable_error`. Additional drift: request used `documents`/`content_b64` instead
of `files`/`content_base64`, omitted `schema`+`prompt`, and the result read
`result_data` instead of `data`.

## Fix

`backend/src/ai/upload/job-service.ts`: aligned the client to the contract
(create body `files`/`content_base64`/`external_job_id` + `schema` + `prompt`; read
`.job_id`; result `.data`; run `{mode:'extraction'}`). Added
`runtime-contract.test.ts` locking the field names so the drift can't regress.
