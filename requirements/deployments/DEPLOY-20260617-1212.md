# DEPLOY-20260617-1212

## Included requirements

- REQ-039 — Rendere tutti i dati ClinicOS interrogabili tramite un Data Gateway agentico (Critical/Architectural) — GitHub Issue #52

## Summary

A typed, secure, read-only **AI Data Gateway** is the single path through which the Agno runtime can
read ClinicOS data — with no direct database access and no SQL surface. Every result carries a
verifiable `SourceReference`; all numeric/temporal filtering is deterministic and server-side; each
call is bound to a user/tenant context the backend enforces; cross-patient search is off by default;
audit records who/what/how-many without any clinical content.

## Changes (backend only)

- `ai/gateway/types.ts` — SourceReference, UserContext, typed IO, GatewayError.
- `ai/gateway/context.ts` — service-token check, user-context parse, tenant pin (single-tenant
  isolation), patient allow-list, role+env-gated cross-patient search.
- `ai/gateway/filters.ts` — deterministic vitals (PA systolic parsing, ranges), allergy/therapy
  match, accent-insensitive text.
- `ai/gateway/sources.ts` — SourceReference builders per domain.
- `ai/gateway/services.ts` — read-only domain services (patients/demographics/allergies/narrative/
  vitals/therapies/diary/documents/appointments/timeline/search/correlate/source-resolve),
  enforcing tenant + patient scope, returning sourced results, auditing each call.
- `ai/gateway/audit.ts` — PHI-free structured audit line.
- `routes/internal-ai.ts` mounted at `/internal/ai` (service-token + user-context gated);
  NO generic SQL/query endpoint.
- tests `ai/__tests__/gateway.test.ts` (11) + `.claude/skills/run-clinicos/req039-gateway-check.mjs`.

## Limitation / next layer

The Agno Python **tool catalog** (Pydantic tools that call these endpoints with the service token +
user context, SOURCE_ONLY agent) is the consuming layer (REQ-039b) — not in this deploy. The gateway
(security + correctness) is complete and deployable now.
