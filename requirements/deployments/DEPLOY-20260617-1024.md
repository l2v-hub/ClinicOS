# DEPLOY-20260617-1024

## Included requirements

- REQ-037 — Riconoscere e filtrare header e footer ripetitivi (Critical) — GitHub Issue #49

## Summary

Repetitive per-page patient headers and page-number footers are now removed from the combined OCR
transcription BEFORE the narrative sections are composed, so a header printed on every page no longer
breaks Anamnesi/Decorso/Terapia continuity or duplicates the anagraphic data. Deterministic,
config-driven, backend-only (no model call, no per-page OCR). The original OCR text is kept
immutable; a cleaned text is what the parser consumes.

## Changes

- backend `ai/sections/header-filter.ts` (new): `filterRepeatedHeaders` — repeated-header detection
  with a position/repetition/label/table-layout confidence score, threshold-gated removal (keeps the
  first occurrence → anagrafica preserved; ambiguous blocks kept + flagged), page-number recovery
  before footer strip, idempotent. Config via `DOCUMENT_HEADER_LABELS`,
  `DOCUMENT_HEADER_REQUIRED_MATCHES`, `DOCUMENT_HEADER_CONFIDENCE_THRESHOLD` (default 0.85).
- backend `ai/upload/job-service.ts`: `runJob` cleans `rawText` → `cleanedRawText` before
  `parseNarrativeFromMarkdown`; stores `cleanedRawText` + `_headerFilter` metadata; self-heal
  re-parse uses the cleaned text.
- tests `ai/__tests__/header-filter.test.ts` (14) + `scripts/req037-evidence.ts` (text artifacts).

## Tests / verification

- backend node:test **112/112**, `tsc` 0 errors.
- live deterministic pipeline artifact: 3 repeated headers + 3 footers → header kept once, 2
  duplicates removed, footers stripped (pages 1-3 recovered), cross-page sections contiguous.
- data smoke: prod `/patients` 200, 19 patients before & after (no regression).

## Limitation

The runtime `document_profiles.process_document` per-page contract pipeline stays unwired because the
AI runtime transcribes all files in one call (no per-page OCR). REQ-037's live outcome is delivered
at the Node layer; the per-page runtime wiring (PDF split + per-page OCR) is tracked separately.

## Deployment

- backend: Railway (deploy-backend.yml). No frontend change (no Vercel deploy needed).
