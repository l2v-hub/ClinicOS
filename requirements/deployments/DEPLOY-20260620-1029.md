# DEPLOY-20260620-1029

## Included requirements

- REQ-041 — Introdurre comandi vocali per aggiungere parametri e modificare dati (Critical / Safety-sensitive) — GitHub Issue #54

## Summary

Adds confirmed voice WRITE-actions to ClinicOS on top of the REQ-039 read gateway and REQ-040 assistant.
Audio is transcribed client-side (Web Speech, it-IT) and never reaches the backend. A spoken command
becomes a deterministic, typed `ActionPlan` (the model never touches the DB); writes always produce a
source-grounded preview and require explicit operator confirmation; ambiguous commands cannot be
confirmed; a double-confirm/retry is idempotent. v1 tools: create_vital_sign, update_patient_demographics
(field whitelist), update_narrative_section (append; originalText immutable), add_diary_note. Deletes,
therapy and allergy changes are refused. Every outcome is audited (source=VOICE, PHI-safe).
