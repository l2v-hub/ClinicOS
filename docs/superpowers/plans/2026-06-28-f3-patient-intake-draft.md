# F3 — PatientIntakeDraft + Transactional Confirm (#125) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`).

**Goal:** A server-persisted `PatientIntakeDraft` that accumulates the in-progress patient intake (manual or import), plus a transactional `confirm` that materializes the draft into `Patient` + all clinical sections in one idempotent transaction.

**Architecture:** New Prisma model `PatientIntakeDraft` (status + JSON `data` + source + audit). A draft service (create / get / patch-autosave / list) behind `/intake/drafts` routes (operator-gated). A `confirmDraft(draftId, payload)` that reuses the existing transactional materialization logic from `confirm-service.ts` (Patient + Cartella + narrative + documents), reading from `draft.data` instead of an import payload — idempotent, full rollback, audited.

**Tech Stack:** Node + Express + Prisma 7 + Postgres. Tests via node test runner (`npm --prefix backend test` / `tsx --test`). Local DB = podman `clinicos-postgres`.

## Global Constraints

- Prisma schema change IS authorized here (EPIC #125 + approved design require `PatientIntakeDraft`). Migration name `add_patient_intake_draft`. Do NOT alter unrelated models.
- Reuse `confirm-service.ts` materialization — do NOT duplicate the patient-create transaction; extract a shared helper if needed.
- Idempotent confirm (re-confirm → same patient, no duplicate); full rollback on error; never a silently partial patient; audit the operator; draft stays `draft` until confirm succeeds, then `confirmed` with `confirmedPatientId`.
- Run migrations locally with the DB up: `podman start clinicos-postgres`; `node node_modules/prisma/build/index.js migrate dev --name add_patient_intake_draft --schema=prisma/schema.prisma`. Regenerate client: `npm run db:generate`.
- Backend type-check authoritative on Vercel/Railway; locally verify with `npm --prefix backend test` (tsx) and `node node_modules/typescript/bin/tsc -p backend/tsconfig.json --noEmit` if needed (RAM permitting).
- Tests via node runner; add files under `backend/src/ai/__tests__/` (existing pattern). Commit after every task.

import { test } from 'node:test';
import assert from 'node:assert/strict';
import { createDraft, patchDraft } from '../../intake/draft-service.js';
import { confirmDraft } from '../upload/confirm-service.js';
import { prisma } from '../../lib/prisma.js';

test('confirmDraft creates patient transactionally + is idempotent', async () => {
  const d = await createDraft({ createdById: 'op-test', source: 'manual' });
  await patchDraft(d.id, {
    anagrafica: { nome: 'IntakeMock', cognome: 'Sintetico', dataNascita: '1970-01-01' },
